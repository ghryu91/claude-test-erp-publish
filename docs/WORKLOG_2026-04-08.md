## 2. LAN 접속(`--host`) 환경 트러블슈팅

`npm run dev -- --host`로 LAN IP(`http://135.1.1.119:5173`)로 접속할 때
발생한 두 가지 이슈를 해결.

### 2-1. API 502 Bad Gateway

- 원인: Vite 프록시는 잘 떠 있지만, 타겟인 `localhost:8000`(FastAPI)에 연결 실패
- 해결:
  1. FastAPI(uvicorn)도 `--host 0.0.0.0`로 실행 필요
     ```bash
     uvicorn main:app --host 0.0.0.0 --port 8000 --reload
     ```
  2. `vite.config.js`의 프록시 타겟을 `localhost` → `127.0.0.1`로 권장
     (Windows에서 IPv6 해석 이슈 방지)
- 메모: `vite.config.js` 변경은 dev 서버 재시작 필요 (HMR 대상 아님)

### 2-2. CopyButton 복사 동작 안 함

- 원인: `navigator.clipboard` API의 **Secure Context 제약**
  - ✅ secure: `https://`, `http://localhost`, `http://127.0.0.1`
  - ❌ insecure: LAN IP(`http://135.1.1.119`), 일반 HTTP
  - insecure context에서는 `navigator.clipboard === undefined`
- 해결: `src/components/ui/copy-button.tsx`
  - `window.isSecureContext`로 분기
  - secure → 표준 `navigator.clipboard.writeText`
  - insecure → 레거시 `document.execCommand('copy')` fallback
    (textarea 임시 생성 → select → copy → 제거)
  - try/catch + console.error로 실패 시 침묵 방지
  - 파일 상단에 secure context 제약사항을 주석으로 명시
