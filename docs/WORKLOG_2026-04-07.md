  - 카테고리 그룹(기본 관리 / 영업 관리 / 운송 관리 / 시스템) 구조
  - 대메뉴 클릭 시 Collapsible로 중메뉴 펼침/접힘
  - `MENU_GROUPS` 배열로 메뉴 구조 중앙 관리
- `src/components/layout/AppLayout.tsx` — 전체 페이지 레이아웃
  - `SidebarProvider` + `AppSidebar` + `SidebarInset` 구성
  - 상단 헤더에 `SidebarTrigger`(토글 버튼) 포함
  - 본문 영역 `max-w-[1400px] mx-auto`로 폭 제한

### 변경
- `src/App.tsx` — 탭 기반 → `BrowserRouter` + `Routes` 라우팅으로 전환
- 메뉴/사이드바 관련 라우트 등록

### 트러블슈팅
- **shadcn/ui Sidebar는 Base UI 패턴**: `asChild` 미지원 → `render={<Link to={...} />}` 패턴 사용
- **Collapsible 화살표 회전**: `data-panel-open`은 Trigger에 붙는 속성. `group/trigger` 네임드 group으로 해결
  - `<SidebarMenuButton className="group/trigger" />` + `group-data-[panel-open]/trigger:rotate-90`
- **레이아웃 너비 과다**: 콘텐츠 영역에 `max-w-[1400px]` 추가하여 이전 폭 복원

## 2. UserListPage 개선

### 데이터 없음 처리
- `company === null` 시에도 회사 정보 카드 영역 유지 + "거래처 정보가 없습니다" 메시지
- `selectedUser === null` 시에도 프로그램 상세 패널 유지 + 안내 메시지

- `ResizableTable` 데이터 없을 때 `colSpan` 셀로 "자료가 없습니다." 표시

## 3. 거래처 화면 신규 구현 (Delphi Yu911 재현)

### 새 파일
- `src/data/customerData.tsx` — 거래처 Mock 데이터 + 컬럼 정의
  - `Customer` 인터페이스 (코드/상호/사업자번호/대표자/주소/전화/팩스/이메일/우편번호/업태/종목/거래구분/비고1,2)
  - `CUSTOMER_DATA` Mock (RM_544 ~ RM_556)
  - `CUSTOMER_COLUMNS` ResizableTable 컬럼 정의

- `src/pages/CustomerPage.tsx` — 거래처 등록 페이지 (1차 버전)
  - 상단 폼 + 우측 작업 버튼(추가/확인/취소/삭제/미리보기/출력/작업종료)
  - 하단 거래처 목록 테이블 (행 클릭 시 폼 로드)
  - `Field` 보조 컴포넌트로 라벨+input 통일

- `src/pages/CustomerRegisterPage.tsx` — 거래처 등록 페이지 (검색형 변형)
  - UserListPage 스타일의 검색 컨트롤 (검색 Dialog + 거래처 코드 직접 조회)
  - 회사 정보 카드 스타일(`border-t-4 border-blue-500`)로 폼 카드 표시
  - 코드 뱃지 + 상호 + CopyButton을 카드 상단에 배치
  - 우측 작업 버튼에 **편집** 버튼 추가

### 편집 Dialog
- `editOpen` / `editForm` state로 별도 편집 모달 관리
- "편집" 버튼 클릭 시 현재 selected 값을 폼에 복사하여 Dialog 오픈
- 코드 필드만 readonly, 그 외 모든 필드(거래구분, 상호, 상호약어, 사업자번호, 대표자, 우편번호, 주소, 업태, 종목, 전화, FAX, 이메일, 비고1, 비고2) 편집 가능
- 저장 시 `customers` 배열과 `selected` 모두 갱신
- `max-h-[85vh] overflow-y-auto`로 긴 폼 스크롤 지원

### 라우팅
- `/customers` → `CustomerPage`
- `/customers/register` → `CustomerRegisterPage`
- 사이드바 "영업 관리 > 거래처 관리"에 두 메뉴 모두 등록
