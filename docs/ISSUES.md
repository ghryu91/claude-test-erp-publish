# ISSUES.md

Delphi → 웹 전환 과정에서 발생할 수 있는 이슈 및 고려사항 정리.

---

## 1. 인증/로그인 체계

- 현재 Delphi 로그인 방식 → 웹에서는 JWT/세션 등 완전히 다른 체계 필요
- 모기업→자회사 전환 시 권한 범위 설계 필요
- 상태: 미착수

## 2. 데이터 동시성 (Concurrency)

- Delphi는 단일 사용자가 데이터 점유 → 충돌 없음
- 웹은 여러 명이 동시에 같은 데이터 수정 가능
- 낙관적 잠금(Optimistic Lock) 또는 비관적 잠금 전략 결정 필요
- 상태: 미착수

## 3. 마이그레이션 전략 (과도기 운영)

- 한 번에 전환 불가 → Delphi + 웹 동시 운영 기간 발생
- 동시 운영 중 데이터 정합성 유지 방안 필요
- 고객사별 전환 순서/일정 계획 필요
- 상태: 미착수

## 4. 오프라인 / 네트워크 이슈

- 건설 현장은 인터넷 불안정 가능성 높음
- Delphi는 로컬 실행이라 문제없지만, 웹은 끊기면 작업 유실
- 임시 저장, 재연결 시 복구 등 대비 필요
- 상태: 미착수

## 5. 인쇄/출력

- ERP 특성상 거래명세서, 세금계산서, 출하전표 등 출력물 다수
- 웹 인쇄는 Delphi 대비 제어가 어려움
- PDF 생성 후 다운로드/인쇄 방식 검토 필요
- 상태: 미착수

## 6. 엑셀 연동

- 건설업체 사용자들은 엑셀 내보내기를 매우 빈번하게 사용
- 프론트에서 CSV/XLSX 다운로드 기능 필수
- 엑셀 업로드(대량 데이터 입력) 가능성도 고려
- 상태: 미착수

## 7. MS SQL 2000 호환성

- ROW_NUMBER() 없음 → 페이징 불가
- 백엔드 메모리 페이징 + Virtual Scroll 조합 필요
- 장기적 DB 마이그레이션 계획 필요
- 상태: 미착수

## 8. 상세 폼 편집 UX 결정 (WinForm → Web)

- 기존 WinForm: 행 선택 시 상세가 바로 편집 가능한 입력 폼
- 웹 전환 시 선택지:
  - (A) 즉시 편집형 유지 — 학습 비용 0, 실수 위험 ↑, dirty 상태 관리 복잡
  - (B) 명시적 편집 모드 — [편집] 버튼 → 폼 활성화 → [저장]/[취소], 무결성↑, 클릭 1회 추가
  - (C) 하이브리드(권장) — 기본 읽기 전용 + [편집] 토글, 더블클릭으로 즉시 편집, 다른 행 이동 시 dirty 경고
- 고려사항:
  - 권한 분리(읽기 전용 사용자) 대응
  - 동시 편집/낙관적 잠금(이슈 2번)과 연계
  - 파워 유저(매일 사용) vs 일반 유저 비중
- 적용 대상: CustomerRegisterPage 포함 모든 등록/조회 화면의 상세 폼
- 상태: 미착수 (방침 결정 필요)
